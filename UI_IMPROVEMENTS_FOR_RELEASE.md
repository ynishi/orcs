# UI改善タスク - リリースに向けて

**作成日**: 2025-11-24
**目的**: 個人ツールとしてのリリース前に、UI/UXの統一性を改善し、ユーザーのミスリードを防ぐ

## 背景

現状、以下の問題がある：
- アイコン体系の混在により視覚的な一貫性が欠如
- スタイルの不統一がユーザーの認知負荷を高める
- 表示の問題（見切れなど）がUXを損なう
- **機能上のミスリードに繋がる可能性がある**

OSSとして公開するため、完璧を目指さないが、基本的なUI統一性は確保したい。

---

## 実装順序

### フェーズ1: 統一性の問題（UI一貫性）

#### 1. アイコンの統一 - SVG系と絵文字系の混在を修正
**優先度**: 高
**理由**: 視覚的な一貫性の基盤。他のタスクにも影響
**ステータス**: ✅ 調査完了

**詳細調査レポート**: `ICON_SYSTEM_INVESTIGATION.md`

**調査結果サマリー**:
- 絵文字使用: 26ファイル（主にNavbar、システムメッセージ）
- @tabler/icons-react使用: 15ファイル（主にアクションボタン）
- 混在による問題: UI一貫性の欠如、テーマカラーとの統合困難

**推奨方針**:
- **Phase 1**: Navbarを優先的に統一（特にペルソナアイコン⭐️の変更）
- **統一方針の選択肢**:
  1. @tabler/icons-reactに統一（推奨・プロフェッショナル）
  2. 絵文字に統一（カジュアル・軽量）
  3. ハイブリッド（UI要素=SVG、テキスト装飾=絵文字）

**次のアクション**:
- [ ] 統一方針を決定
- [ ] Personasアイコンを`IconMask`/`IconUsers`または🎭/😊に変更
- [ ] NavbarIcon コンポーネントの型定義変更（必要に応じて）

  - どこに表示されているか

  - 削除した場合の影響

#### 7. ステータスバーの文字見切れ対応 - アイコン+ツールチップ形式に変更

**詳細**:
- 現状: ステータスバーで文字が見切れている
- 目標: デフォルトはアイコン表示、ホバーでツールチップに詳細表示
- 実装項目:
  - アイコン化するステータス情報の特定
  - ツールチップコンポーネントの実装
  - 適切なアイコン選択

## 実装方針

1. **まず調査**: 各タスクの該当箇所を特定し、影響範囲を把握
2. **統一性優先**: フェーズ1を先に完了させる
3. **段階的リリース**: 完璧を目指さず、段階的に改善
4. **ドッグフーディング**: 普段使いで気になる点を都度修正

---

## 除外項目

以下は**今回対応しない**：
- 細かいバグ修正（製品でもバグはある、OSSなので気にしない）
- パフォーマンス最適化（必要になったら対応）
- 大規模なリファクタリング
